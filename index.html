How it Works

The goal is not to build a smarter model, but to build a smarter inference engine that makes existing models faster, cheaper, more consistent, and able to handle much longer reasoning sessions by remembering and reusing work instead of starting from scratch every time.

Neura builds a task plan before invoking AI — deciding what to read, recall, run, and generate.

Four cooperating stores keep work reusable — click a card to inspect each subsystem.

Stores durable knowledge — not chat history. Embeddings find likely memories first; graph edges then expand to nearby related ones.

  • User preferences
  • Project knowledge
  • Decisions
  • Relationships
  • Graph connections
  • Embedding search
  • Cascade retrieval

Long conversations are compressed, not discarded. Compact references stay in the prompt; exact text is stored locally and restored with .ctx_get.

Avoids repeating work — if a result is already known, return it; otherwise execute and save.

Not the main AI. A small local helper model for memory hygiene — extract, merge, summarize, verify relevance, and detect contradictions. After each response it may save durable memories locally.

  • Memory extraction
  • Relevance verification
  • Summarization
  • Consolidation
  • Contradiction detection

Neura performs real work instead of guessing. The planner decides when tools should be invoked.

↓

Deterministic problems do not require an LLM — math, JSON, YAML, regex, dates, and unit conversions run here.

  • Math
  • JSON
  • YAML
  • Regex
  • Dates
  • Unit conversions
↓

AI is only invoked after planning, retrieval, and tooling. Neura picks the best path for the task and hardware.

↓

Every result is checked before returning — tools, cache, files, tests, confidence, and evidence.

  • Tool validation
  • Cache validation
  • File verification
  • Test results
  • Confidence scoring
  • Evidence checking
↓

Only after planning, retrieval, tooling, reasoning, and verification does Neura return the final response.
